# Descripción del Proyecto y Requisitos Iniciales

Este documento define la base conceptual del proyecto y enumera las herramientas, librerías y estructuras requeridas para comenzar la implementación de la IA.

---

## 📝 Descripción del Proyecto: "NetMind-802"

**NetMind-802** es un modelo de inteligencia artificial multimodal pequeño, especializado, y diseñado desde cero en Python. Su propósito principal es actuar como un **asistente de diagnóstico y seguridad telemática** para redes cableadas e inalámbricas bajo estándares **IEEE 802**.

### 🌟 Características Clave:
*   **Analítico y Multimodal:** Puede recibir texto (logs de consola, salidas de comandos, descripciones de problemas) e imágenes (topologías de red, capturas de pantalla de Wireshark).
*   **Resolución en Cascada:** Sus respuestas siguen una secuencia estructurada y estricta: **Diagnóstico de Falla $\rightarrow$ Posibles Causas $\rightarrow$ Comandos de Solución**.
*   **Enfoque Multiplataforma:** Proporciona soluciones aplicables tanto a sistemas operativos de host (**Windows PowerShell/CMD**, **Linux**) como a sistemas operativos de red (**Cisco IOS, Junos, RouterOS**).
*   **Ejecución Local:** Diseñado para ser lo suficientemente ligero como para entrenarse y ejecutarse en una computadora personal, sin necesidad de conectarse a APIs externas (máxima privacidad y offline).

---

## 🛠️ Requisitos para Iniciar (Omitiendo Python y VS Code)

Para comenzar el desarrollo de **NetMind-802**, necesitarás preparar tu entorno con los siguientes elementos:

### 1. Librerías de Python Requeridas (Dependencias)
Necesitaremos instalar las siguientes librerías de procesamiento numérico, visión por computadora y aprendizaje profundo:

*   **`torch` (PyTorch):** El motor principal para construir y entrenar las capas de nuestra red neuronal desde cero.
*   **`torchvision`:** Para el manejo y preprocesamiento de imágenes (redimensionamiento, normalización) y uso de arquitecturas CNN de base.
*   **`tokenizers` (de Hugging Face):** Para crear y entrenar nuestro tokenizador BPE (Byte Pair Encoding) personalizado de forma eficiente.
*   **`pillow` (PIL):** Para cargar y manipular los archivos de imagen de topologías y capturas.
*   **`numpy`:** Para la manipulación de arreglos numéricos y matrices.

> [!TIP]
> Puedes instalar todo ejecutando en tu terminal:
> ```bash
> pip install torch torchvision tokenizers pillow numpy
> ```
********************************************************************************************************************************
### 2. Recursos de Hardware
*   **CPU:** Cualquier procesador moderno (Intel Core i5/Ryzen 5 o superior). El entrenamiento de un modelo pequeño tardará un poco más en CPU, pero es perfectamente viable para aprender.
*   **GPU (Opcional pero recomendado):** Si tienes una tarjeta gráfica NVIDIA con soporte para CUDA, el entrenamiento será considerablemente más rápido.

### 3. Estructura de Carpetas Sugerida para el Proyecto
Para que tu código sea modular y limpio, te recomiendo crear la siguiente estructura en tu directorio de trabajo:

```text
netmind-802/
│
├── data/
│   ├── dataset.json            # Base de conocimientos (casos, problemas y soluciones)
│   └── images/                 # Carpeta para capturas de red, topologías, etc.
│
├── src/
│   ├── __init__.py
│   ├── tokenizer_builder.py    # Código para entrenar y guardar el tokenizador
│   ├── dataset_loader.py       # Carga de datos y preparación para PyTorch (DataLoaders)
│   ├── model_architecture.py   # Arquitectura de la red neuronal (CNN + Transformer)
│   ├── train.py                # Bucle de entrenamiento y optimización
│   └── test_inference.py       # Código para probar la IA con nuevas preguntas
│
├── models/                     # Carpeta donde guardaremos los pesos (.pth) de la IA entrenada
│
└── procesos/            # Lista de dependencias del proyecto
```

### 4. Datos Iniciales (Corpus)
Necesitaremos escribir un pequeño corpus de texto que contenga:
*   Documentación de comandos (Cisco, PowerShell, comandos de red de Linux).
*   Un archivo de texto plano con términos telemáticos (ej. *VLAN, trunk, STP, IEEE 802.1X, DHCP snooping, SSID, WPA3, handshake, CSMA/CD*).
Este corpus servirá para que el tokenizador aprenda las "palabras" clave antes de entrenar la IA.
